A flexible, project-centered setup for data science

In this tutorial you will learn how to link Sublime Text 3 to an environment from Anaconda/Miniconda and execute Python with SublimeREPL. You might already have existing projects with different conda virtual environments - no worries. This setup can be easily applied to any project that you already have.

Photo by Keira Burton from Pexels

TL;DR

  1. Open Sublime Text 3 and add the packages package control, SublimeREPL and Project Manager. Shortcut: CTRL+SHIFT+p, type install > Package Control: Install Package > [package name])
  2. Note the name of your conda env or create a new one conda create --name example_env python=3.8 -y
  3. Having C:\Program Files\Sublime Text 3\ or wherever subl.exe is in your system environment variables, open cmd in the project directory and open Sublime with subl .
  4. Add a new project to Project Manager with the command palette via CTRL+SHIFT+p > Project Manager: Add new project
  5. CTRL+SHIFT+p type browse and select Preferences: Browse packages
  6. From the opened explorer, navigate to SublimeREPL/config/Python, copy the file Main.sublime-menu and go back to C:\Users\[YOURNAME]\AppData\Roaming\Sublime Text 3\Packages\.
  7. Within the User folder, create SublimeREPL\config\Python directories and paste Main.sublime-menu into C:\Users\[YOURNAME]\AppData\Roaming\Sublime Text 3\Packages\User\SublimeREPL\config\Python. Rename Main.sublime-menu into example_env.sublime-menu and open it with Sublime.
  8. Search for the block where "id": "repl_python_run",, replace "cmd": ["python", "-u", "$file_basename"], with "cmd": ["C:\\Users\\[YOURNAME]\\[PATHTOCONDA]\\envs\\example_env\\python.exe", "-u", "$file_basename"], and save.
  9. CTRL+SHIFT+p find Project: edit project add and save:
"build_systems":
    [
        {
            // 1. name the build system meaningfully
            "name": "Conda Python example_env REPL", 
            "target": "run_existing_window_command", 
            "id": "repl_python_run",
            // 2. Replace example_env with any name you defined in (5) above
            "file": "config/Python/example_env.sublime-menu", 
        }
    ],
  1. You should be able to see Conda Python example_env REPL among Tools>Build System and can use the Python REPL with the Python of your conda env. Repeat the process for each project.
  2. Switch easily between projects with CTRL+ ALT + p and do not worry about the conda environment. The setting is project specific. I hope this expedites your workflow. Have fun with all your future projects!

This post was inpspired by community efforts on Stackoverflow. My approach avoids that the Sublime build-system menu will be filled with every virtual environment you link to REPL. Furthermore, the solution here is project-centered or environment-centered instead of a global setting.

Problem statement

The more projects you have, the more virtual environments you create, the more you collaborate with others - the more important are project-specific settings. I experienced this recently when three projects ran in parallel and I had to frequently switch between those. Switching between projects and build systems disrupted my workflow. But I love the tools I have: Sublime, Anaconda and Sublime REPL. This article suggests a way to smoothly integrate all three for a quick & stable project setup.

What could be an ideal Python project setup?

  • quickly load files with a lightweight text editor –> Sublime Text 3
  • switch effortlessly between projects –> Sublime Project Manager
  • solve dependency issues with virtual environments for each project –> Anaconda + Sublime setup
  • have an interactive command line at our fingertips –> Sublime REPL

This setup helped me to quickly switch between projects and environments by using the Project Manager. I hit CTRL+ALT+p, select the other project, and most importantly, have the build system switched as well. A few steps have to be followed, but it is worth it. Hopefully you find this useful, as well.

Getting started

Install the following if you do not have it already:

  1. Sublime Text 3: https://www.sublimetext.com/3
  2. Sublime Package Control: https://packagecontrol.io/installation
  3. Project Manager: https://packagecontrol.io/packages/ProjectManager
  4. Sublime REPL: https://packagecontrol.io/packages/SublimeREPL
  5. Anaconda/Miniconda: https://docs.conda.io/en/latest/miniconda.html

This tutorial assumes Windows 10, but should similarly work on Linux. I recommend using Miniconda, see why on Reddit

Solution

We installed everything from the above list and have a conda environment named example_env. The first step is to center Sublime around the project root folder. In this way, all shortcuts like jumping to a file with CTRL+p relate to the project root ./. Then we initialize the Project Manager.

Let us open a new instance of Sublime at the project root like C:\Users\[YOURNAME]\[PATH-TO-PROJECT]\[PROJECTNAME]. I have Sublime added to my environment variables (find a short how-to below) and open a terminal in the project root directory which is C:\Users\philipp\projects\Sublime project setup. Access the address bar with CTRL+l, type cmd to open command, ↵ Enter. Run subl . (only works if you added subl.exe to your Windows environment variables). An empty Sublime window opens. There you go!

Now we turn towards the Project Manager. Add a new project to Project Manager with the command palette via CTRL+SHIFT+p > Project Manager: Add new project. Check out the project setting file with CTRL+SHIFT+p > Project Manger: Edit project. Sublime stores your project settings here, including project name and path:

Next, we define a build system which takes the python.exe of the project conda env, example_env. Note the name of your conda environment. For example, I created an environment with conda create --name example_env python=3.8 -y for this tutorial. So, I use example_env as an environment here.

  1. CTRL+SHIFT+p type browse and select Preferences: Browse packages. The browser opens in C:\Users\[YOURNAME]\AppData\Roaming\Sublime Text 3\Packages\.
  2. From here, navigate to SublimeREPL/config/Python and copy the file Main.sublime-menu
  3. Go back to C:\Users\[YOURNAME]\AppData\Roaming\Sublime Text 3\Packages\, open User and create the following folders: SublimeREPL\config\Python.
  4. Now you are in C:\Users\[YOURNAME]\AppData\Roaming\Sublime Text 3\Packages\User\SublimeREPL\config\Python and paste the Main.sublime-menu into the directory.
  5. Rename Main.sublime-menu into example_env.sublime-menu and open it with Sublime
  6. Important step: Search for the block where "id": "repl_python_run",, replace "cmd": ["python", "-u", "$file_basename"], with "cmd": ["C:\\Users\\[YOURNAME]\\[PATHTOCONDA]\\envs\\example_env\\python.exe", "-u", "$file_basename"], and save. Note: Replace the path with any path that leads to the python.exe of your conda environment. In my case this is C:\\Users\\philipp\\Miniconda3\\envs\\example_env\\python.exe.
  7. Open the command palette with CTRL+SHIFT+p find Project: edit project and select the project name. Now we define the build systems that use the Python of our conda environment, example_env. Copy-paste the following snippet and replace example_env with the name of your environment. Important: There are 2 replacements in total.
{
    "build_systems":
    [
        {
            // 1. name the build system meaningfully
            "name": "Conda Python example_env REPL", 
            "target": "run_existing_window_command", 
            "id": "repl_python_run",
            // 2. Replace example_env with any name you defined in (5) above
            "file": "config/Python/example_env.sublime-menu", 
        }
    ],

    "folders":
    [
        {
            "binary_file_patterns":
            [
            ],
            "file_exclude_patterns":
            [
            ],
            "folder_exclude_patterns":
            [
            ],
            "name": "example project",
            // already filled after Project Manager: Add project
            "path": "C:\\Users\\[YOURNAME]\\Projects\\example_project"
        }
    ]
}
  1. Save the file, go to Tools/Build systems in the menu bar. Here you should see your project-specific build systems and select it

Usage

This setup reveals its strength when you work on several projects in parallel. You can (i) switch between projects in no time with CTRL+ ALT + p and (ii) have the related virtual environment among your build systems.

To set up a new project, start out with subl . from the command line and repeat all steps from the Solution section. Doing this around two times, it will come natural in <1 min.

I am sure there are other approaches and project setups which I not yet figured out. So, I appreciate any tips and best practices for project settings in Sublime, REPL and Anaconda.

If there is someone who wants to automate the process and implements a project-based build system for Sublime REPL - I would be grateful! I think many others would benefit from this. It would link the best tools from the Sublime-Anaconda and REPL-world.

Launch Jupyter Notebook or Jupyter lab with a double click

Use a batch file to quickly open a Jupyter lab or notebook instance. Note: The following assumes Win 10. Furthermore, if you have Anaconda installed, replace the miniconda with anaconda in the paths.

  1. Navigate to your Anaconda/Miniconda folder and open the Scripts directory. For example, in my case this is C:\Users\yourname\miniconda3\Scripts.
  2. Create a new file activate_jupyterlab.bat, open with a text editor and insert:
@set "_args1=%1"
@set _args1_first=%_args1:~0,1%
@set _args1_last=%_args1:~-1%
@set _args1_first=%_args1_first:"=+%
@set _args1_last=%_args1_last:"=+%
@set _args1=

@if "%_args1_first%"=="+" if NOT "%_args1_last%"=="+" (
    @CALL "%~dp0..\condabin\conda.bat" activate
    @GOTO :End
)

@CALL "%~dp0..\condabin\conda.bat" activate %*

:End
@set _args1_first=
@set _args1_last=
@CALL jupyter lab

If you prefer jupyter notebook, change @CALL jupyter lab to @CALL jupyter notebook and rename activate_jupyternotebook.bat accordingly.

Go to the project directory and a folder where you want to open Jupyter. For example, I store create a new batch file like start_jupyterlab.bat and insert

%windir%\System32\cmd.exe "/K" C:\Users\[YOURNAME]\miniconda3\Scripts\activate_jupyterlab.bat project1_env

Add sublime.exe to environment variables

Shortcut for Windows users: Hit , type env and select Edit environment variables for your account from the search results.

Then click Path > Edit > New > C:\Program Files\Sublime Text 3\ > OK. Replace C:\Program Files\Sublime Text 3\ with the directory where you find subl.exe. Advanced project setup for data science

Find your way easily: organized folder structure with cookiecutter

Some argue that each data science project is unique and the project setup should reflect this. It might be no issue if you work on two or three projects where you know their directory structure by heart. This approach is often found within academia. However, a more sustainable approach is to build your project around proven guidelines that emerged from the community. It makes your projects easy to share, explain to others and enables cooperation. Think about others when you set up projects. They should have an easy time finding their way and understand how you organized a data science project. It can be summarized in one principle:

Rely on existing and proven project structures. If you deviate from it, make the structure self-explanatory. Take the perspective of someone who has 30 seconds to grasp your awesome project on Github. Minimize the gap between what others expect of your project structure and your individual requirements.

The easiest way to do this is with Cookiecutter data science. Start out with their folder template, adjust it and delete anything your project does not need. Another template I love is Cookiecutter modern data science.
